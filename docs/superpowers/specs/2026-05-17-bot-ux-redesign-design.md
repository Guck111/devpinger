# Bot UX Redesign — Design Spec

**Дата:** 2026-05-17
**Статус:** Drafted, awaiting review

## Контекст

Сейчас в `apps/server/src/bot` зарегистрировано 15 slash-команд (12 в Telegram menu + 3 скрытых: `/status`, `/notify_self`, `/forget_event`). У пользователя нет:
- описания бота в Telegram-карточке (`setMyDescription` / `setMyShortDescription` не вызываются);
- внятного welcome — текущее `start.welcome` лишь приветствует и просит «подключи сервис»;
- инструкций по использованию (`/help` — плоский список команд, без пояснений и пошагового сценария);
- визуальной навигации — discoverability через кнопки отсутствует, кроме inline-меню `buildStartMenu` в `index.ts:65-91`.

В `packages/i18n/src/messages/{ru,en}/bot.json` уже задизайнен, но не реализован, hub-меню (ключи `hub.*`, `mainKeyboard.*`) — направление было нащупано раньше, но брошено.

## Цель

Подготовить бота к заливке трафика с лендинга. После клика «попробовать в Telegram» новый пользователь должен:
1. Видеть в карточке бота, что он делает (до нажатия Start).
2. Получить пошаговый wizard, ведущий к рабочему состоянию (подключение → выбор репо → ожидание событий).
3. После онбординга — иметь постоянную навигацию через reply-keyboard, не запоминая команды.
4. Опытный пользователь сохраняет slash-команды как shortcuts.

## Дизайн

### 1. Telegram BotFather metadata

Установить при старте `apps/server/src/bot/index.ts` (рядом с `setMyCommands`):

- `setMyShortDescription` (виден в поиске и share-карточке): `"Уведомления GitHub/Jira в Telegram с действиями одной кнопкой."` (en аналог).
- `setMyDescription` (виден в «What can this bot do»): 200-400 символов — описание сценария, перечисление поддерживаемых событий и действий.
- Локалить как и `setMyCommands` (через `language_code: "ru"`).

### 2. Persistent reply-keyboard

Раскладка 2×2, всегда показывается под полем ввода после welcome / в основном меню:

```
📡 Подключения  |  🔔 События
⚙️ Настройки    |  ❓ Помощь
```

Каждая кнопка отправляет соответствующий текст; обработчик `bot.hears(/^📡 Подключения$/)` и т.п. открывает inline-меню раздела. Это работает в DM (где бот и живёт).

Поведение:
- После `/start` (для всех состояний) клавиатура устанавливается через `reply_markup: ReplyKeyboard`.
- При смене языка кнопки локализуются (i18n-driven), для матчинга через `bot.hears` использовать i18n-ключи + регулярки по всем локалям.

### 3. Inline-меню разделов (hub)

**📡 Подключения**
- 🐙 GitHub — если не подключён: кнопка `[🔗 Подключить GitHub]` (OAuth URL). Если подключён: `[✅ Подключён как @login]` (открывает submenu «📁 Репозитории» + «🔌 Отключить»).
- 🟦 Jira — то же.
- Кнопка «Отключить» удаляет запись из `connections` и предупреждает, что бот больше не будет получать события (отзыв через GitHub/Jira UI требуется отдельно — об этом писать в подсказке).

**🔔 События**
- 📜 Последние события (`recentEvents` → текст)
- 📊 Статистика (`userStats` → текст)
- 🔕 Мьюты (`listMutes` → inline-список с удалением)

**⚙️ Настройки**
- 🌐 Язык (текущий: ru/en) — открывает inline c выбором.
- 🔔 Уведомления — тогл «📢 Свои действия: ВКЛ/ВЫКЛ» (бывший `/notify_self`).
- 👤 Аккаунт — submenu:
  - 📤 Экспорт данных (бывший `/export`).
  - 🗑 Удалить аккаунт (бывший `/unsubscribe`, с двухшаговым подтверждением — оно уже есть в `account.ts`).

**❓ Помощь**
- 📖 О боте (1 абзац, что бот делает + пример события).
- 🚀 Быстрый старт (3 шага текстом).
- 💬 FAQ (5-7 частых вопросов).
- ✉️ Поддержка (контакт / @-канал).

Навигация: каждое подменю имеет кнопку `← Назад`. Корень hub — кнопка `✖ Закрыть` (удаляет inline-сообщение, reply-keyboard остаётся).

### 4. Onboarding wizard для нового юзера

Триггер: `/start` без аргументов **и** `connections.length === 0` для текущего пользователя.

4 отдельных reply-сообщения подряд:

**Сообщение 1 — Welcome:**
> Привет, {username}! Я DevPinger.
>
> Собираю уведомления из **GitHub** и **Jira** в один Telegram-чат и даю действия одной кнопкой: ✅ Approve, 💬 Comment, 🚀 Merge, 🔕 Mute.
>
> Без открытых вкладок и шумных каналов — только то, что важно.

**Сообщение 2 — Шаг 1/3:**
> **Шаг 1 из 3 — подключи провайдер**
>
> [🐙 Подключить GitHub] (OAuth-кнопка)
> [🟦 Подключить Jira]
>
> Достаточно одного, можно оба позже.

**Сообщение 3 — Шаг 2/3** (присылается **после** возврата из OAuth, реагируя на deep-link `?start=connected_github` / `connected_jira` — этот хук уже есть в `index.ts:96-112`):
> ✅ {Provider} подключён как @{login}.
>
> **Шаг 2 из 3 — выбери, за чем следить.**
>
> [📁 Открыть список репозиториев] (для GitHub) / [📋 Открыть список проектов] (для Jira)

**Сообщение 4 — Шаг 3/3** (присылается после первого `repos:add` / `proj:add` callback'а **в контексте онбординга** — определяем по флагу в БД):
> 🎉 Готово! {fullName} подключён.
>
> **Шаг 3 из 3 — жди первое событие.**
>
> Управление — кнопки внизу. Если что — `❓ Помощь`.
>
> _(внизу появляется reply-keyboard 2×2)_

Для отслеживания состояния онбординга добавить поле `users.onboardingCompletedAt: timestamp | null`. После шага 3 — выставляется. После — wizard больше не повторяется.

### 5. Adaptive `/start` для существующего юзера

Если `connections.length > 0` **или** `onboardingCompletedAt !== null`:

```
С возвращением, {username}!

📡 {N} {репо|репо|репо} · 📊 {M} событий за 7 дней

Выбирай раздел внизу или жми /help.
```

+ reply-keyboard 2×2.

### 6. Контекстные follow-up

- После `repos:add` (первого) **вне** онбординга: `«✅ {fullName} подключён. Первые события — в ближайшие минуты.»`
- После первого реального события (определяется по `events.count() === 1` для юзера): однократная подсказка `«👋 Первое событие! Кнопки выше — это действия.»`
- После настройки мьюта: `«🔕 Мьют добавлен. Управление — Настройки → 🔕 Мьюты.»`

Follow-up для «первого события» отправляется из worker'а после `notification.sent`. Это новый код — добавить флаг `users.firstEventNotifiedAt` для идемпотентности.

### 7. Refactor `/notify_self`

Сейчас в `bot/index.ts:245-261` команда защищена `telegramId !== env.ADMIN_TELEGRAM_ID`. Убираем guard, переподключаем как `settings:notify_self:toggle` callback в hub'е. Команду `/notify_self` оставляем работать как скрытый shortcut (без admin guard).

### 8. Slash-команды в Telegram menu

После редизайна `commands-menu.ts` содержит:

| Команда | Описание |
|---|---|
| `/start` | Главное меню |
| `/help` | Что делает бот, как им пользоваться |
| `/repos` | Репозитории GitHub |
| `/projects` | Проекты Jira |
| `/mutes` | Управление мьютами |
| `/recent` | Последние 20 событий |
| `/stats` | Сводка активности |
| `/lang` | Сменить язык |
| `/cancel` | Отменить текущий шаг |

**Скрытые** (работают, но не в menu): `/export`, `/unsubscribe`, `/forget_event`, `/sources`, `/notify_self`, `/status`. `/sources` устаревает — содержимое уходит в Помощь → О боте.

### 9. Переписать `/help`

Сейчас — плоский список команд. Меняем на структурированный текст с разделами и примерами:

```
🤖 Что я делаю
Собираю события из GitHub и Jira и шлю их сюда — c кнопками действий.

📡 Поддерживаемое
GitHub: PR, issue, ревью, комментарии, релизы, упавшие workflow.
Jira: issues, переходы статусов, комментарии, упоминания.

🚀 Как пользоваться
1. /start — подключи провайдер.
2. /repos или /projects — выбери, за чем следить.
3. Жди первое событие — там будут кнопки действий.

🔕 Лишний шум
/mutes — отключи целый источник, репо, проект или тип события.

⚙️ Управление
Кнопки внизу: 📡 Подключения · 🔔 События · ⚙️ Настройки · ❓ Помощь
```

## Изменения в коде

**Новые файлы:**
- `apps/server/src/bot/hub.ts` — обработчики reply-keyboard (`bot.hears`) и inline callback-ы для разделов/подразделов.
- `apps/server/src/bot/onboarding.ts` — wizard и helpers (`isOnboarding`, `markOnboardingStep`).
- `apps/server/src/bot/metadata.ts` — `registerBotMetadata(api)` для description / short_description (вызывается рядом с `registerBotCommands` в `index.ts`).
- `apps/server/src/bot/help.ts` — выделить `/help` в отдельный файл (текст длинный, форматирование).

**Модифицируемые файлы:**
- `apps/server/src/bot/index.ts` — adaptive `/start`, monolith разобрать минимально (handler-ы зарегистрировать из новых модулей).
- `apps/server/src/bot/commands-menu.ts` — сокращённый список (см. выше), новые описания команд.
- `apps/server/src/bot/account.ts` — добавить callback-ы для «Настройки → Аккаунт → Экспорт/Удалить».
- `packages/i18n/src/messages/{ru,en}/bot.json` — массивные правки: добавить hub.*, onboarding.*, help.* (расширенный); удалить устаревшие; локализовать metadata.
- `packages/db/src/schema/*` — добавить `users.onboardingCompletedAt` и `users.firstEventNotifiedAt`.

**Миграция БД:** drizzle migration с двумя `timestamp("...")` колонками, nullable.

**Удаление admin guard** на `/notify_self` (см. п.7).

## Тесты

- **E2E onboarding** (apps/server/test/e2e): mock Telegram, прогон 4-шагового wizard'а от welcome до `firstEventNotifiedAt`.
- **E2E hub navigation**: проверка переходов по разделам, корректность inline-кнопок и подсказок.
- **E2E adaptive /start**: новый юзер vs существующий — разные сообщения и keyboard.
- **Integration `/help`**: рендер текста, не падает на отсутствующих i18n-ключах.
- **Unit metadata**: `registerBotMetadata` вызывает `setMyDescription` / `setMyShortDescription` для en и ru с правильными аргументами (mock `api`).

## Out of scope

- Удаление scope существующих slash-команд (оставляем для backward compat).
- Изменения в формате уведомлений (`destinations-telegram/formatEvent`).
- Перевод бота на другие языки кроме en/ru.
- Кастомные клавиатуры для группового чата (бот живёт в DM).
